# ADR-001 — Strategia rozwoju BAŁTYCKIE OS

Status: Accepted
Data: 2026-06-27

## Kontekst
Istnieje działająca aplikacja e-Bałtyckie rozwijana w Replit. System jest już używany operacyjnie i zawiera wartościową logikę biznesową.

## Decyzja
Projekt rozwijamy ewolucyjnie.

Nie tworzymy nowego systemu od zera.

Każda nowa funkcjonalność przechodzi przez proces:

1. Analiza
2. Projekt
3. Implementacja
4. Dokumentacja

Kod źródłowy pozostaje w GitHub.
Replit pozostaje środowiskiem implementacji.
GitHub jest źródłem prawdy dla dokumentacji technicznej.

## Uzasadnienie
- mniejsze ryzyko,
- zachowanie działającego systemu,
- szybszy rozwój,
- możliwość ciągłego wdrażania zmian,
- pełna historia decyzji.

## Konsekwencje
Każda większa decyzja architektoniczna będzie dokumentowana jako kolejny ADR.
